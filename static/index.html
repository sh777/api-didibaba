DiDiBaBa API

Generate TradingView chart screenshots via a simple REST API. Supports stocks, crypto, forex, and any TradingView symbol.

https://api.didibaba.ai

Introduction 👋

Welcome to the DiDiBaBa API. This service provides programmatic access to TradingView chart screenshots for use in trading bots, analysis tools, and automated reporting.

The API is built on FastAPI and uses Playwright + Chromium to render live TradingView charts with your saved layouts, indicators, and themes intact.

Interactive API docs are also available at /docs (Swagger UI) and /redoc (ReDoc).

Base URL 🔗

https://api.didibaba.ai

Authentication 🔒

This is a private API. No public API key is required — access is controlled at the network/deployment level. If you need access, contact the DiDiBaBa team.

GET /chart/image

Generate a TradingView chart screenshot and return it as a PNG image.

GET /chart/image Returns a PNG chart image

Query Parameters

Parameter Type Required Default Description
symbol string required — TradingView ticker symbol. e.g. NASDAQ:AAPL, BINANCE:BTCUSDT, SSE:600519. Short forms like AAPL or BTC are also accepted and auto-normalized.
interval string optional 1D Chart time interval. See Intervals for valid values.
chart_type string optional default Preset chart layout. One of default, btc, fabio. Ignored if chart_id is set. See Chart Types.
chart_id string optional — TradingView saved layout ID (e.g. Pmtyn6fy). Takes priority over chart_type when provided. Use this to reference any custom saved layout.
width integer optional 1920 Output image width in pixels. Range: 400–3840.
height integer optional 1080 Output image height in pixels. Range: 300–2160.

Responses

200 OK

Returns a PNG image (image/png). The Content-Disposition header includes a filename like AAPL-1D.png.

400 Bad Request

Invalid parameter value (e.g. unsupported interval).

500 Internal Server Error

Chart generation failed (TradingView session issue, Playwright timeout, etc.).

Chart Types

Preset chart layouts available via the chart_type parameter. For full control, use chart_id to reference any TradingView saved layout directly.

default

General purpose chart. Used for stocks, indices, and most assets.

chart_type=default

btc

Bitcoin-optimized layout with crypto-specific indicators.

chart_type=btc

fabio

Custom strategy layout (Fabio Trader).

chart_type=fabio
Using custom layouts: Find the layout ID in your TradingView chart URL — e.g. tradingview.com/chart/Pmtyn6fy/ — and pass it as chart_id=Pmtyn6fy.

Examples 🧪

AAPL daily chart (default layout)

GET https://api.didibaba.ai/chart/image?symbol=NASDAQ:AAPL&interval=1D

BTC weekly chart (btc layout)

GET https://api.didibaba.ai/chart/image?symbol=BINANCE:BTCUSDT&interval=1W&chart_type=btc

A-share (600519 茅台) daily chart

GET https://api.didibaba.ai/chart/image?symbol=SSE:600519&interval=1D

Custom layout by chart_id

GET https://api.didibaba.ai/chart/image?symbol=NASDAQ:TSLA&interval=4h&chart_id=dVrMW7wa

cURL

curl -o chart.png \
  "https://api.didibaba.ai/chart/image?symbol=NASDAQ:AAPL&interval=1D"

Python

import requests

resp = requests.get(
    "https://api.didibaba.ai/chart/image",
    params={
        "symbol": "NASDAQ:AAPL",
        "interval": "1D",
        "width": 1920,
        "height": 1080,
    },
    timeout=90,
)
resp.raise_for_status()

with open("chart.png", "wb") as f:
    f.write(resp.content)

JavaScript (fetch)

Error Codes

StatusMeaningCommon Cause
200 OK PNG image returned successfully.
400 Bad Request Invalid interval or parameter out of range.
422 Unprocessable Entity Missing required parameter (symbol).
500 Internal Server Error TradingView session expired, Playwright timeout, or rendering error.
Timeout note: Chart rendering can take up to 60 seconds for complex layouts or slow network conditions. Set your HTTP client timeout to at least 90s.
